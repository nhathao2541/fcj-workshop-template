## Introduction

In this chapter, we will deploy the entire backend infrastructure of Smart Notes API using **AWS SAM**—including Amazon DynamoDB, Amazon S3, AWS Lambda, Amazon API Gateway, and a least-privilege IAM Role—all defined in a single file: `template.yaml`.

This follows the **Infrastructure as Code (IaC)** approach: no manual configuration through the AWS Console. Every infrastructure change is managed through code, can be reviewed with Git, and can be redeployed consistently across any AWS account.

---

## Create the Amazon DynamoDB Table

```yaml
NotesTable:
  Type: AWS::DynamoDB::Table
  Properties:
    TableName: !Sub "Notes-${StageName}"
    BillingMode: PAY_PER_REQUEST
    AttributeDefinitions:
      - AttributeName: id
        AttributeType: S
    KeySchema:
      - AttributeName: id
        KeyType: HASH
    TimeToLiveSpecification:
      AttributeName: expiresAt
      Enabled: true
```

`PAY_PER_REQUEST` (On-Demand) is chosen because the traffic of a personal note-taking application is unpredictable. You only pay for the requests you actually make instead of paying for provisioned capacity that may remain unused.

---

## Enable DynamoDB Point-in-Time Recovery (PITR)

The `TimeToLiveSpecification` above only handles the **automatic deletion** of expired notes in the recycle bin. It does **not** protect against accidental deletion or modification of active notes caused by users or application bugs.

To provide an additional layer of protection, enable **Point-in-Time Recovery (PITR)**. DynamoDB continuously backs up table changes, allowing you to restore the table to **any point within the last 35 days** without implementing your own backup solution.

Add the following configuration directly below `TimeToLiveSpecification`:

```yaml
NotesTable:
  Type: AWS::DynamoDB::Table
  Properties:
    TableName: !Sub "Notes-${StageName}"
    BillingMode: PAY_PER_REQUEST
    AttributeDefinitions:
      - AttributeName: id
        AttributeType: S
    KeySchema:
      - AttributeName: id
        KeyType: HASH
    TimeToLiveSpecification:
      AttributeName: expiresAt
      Enabled: true
    PointInTimeRecoverySpecification:
      PointInTimeRecoveryEnabled: true
```

PITR does not incur a fixed monthly cost. Charges are based only on the amount of data stored, which is typically minimal for a personal note-taking application. In return, it provides a reliable recovery mechanism for accidental data loss.

To restore the table, use the AWS CLI:

```bash
aws dynamodb restore-table-to-point-in-time \
  --source-table-name Notes-dev \
  --target-table-name Notes-dev-restored \
  --restore-date-time "2026-07-20T10:00:00Z"
```

## Create the Amazon S3 Bucket (Private Image Storage)

```yaml
NotesBucket:
  Type: AWS::S3::Bucket
  Properties:
    BucketName: !Sub "${BucketNamePrefix}-${AWS::AccountId}-${StageName}"
    PublicAccessBlockConfiguration:
      BlockPublicAcls: true
      BlockPublicPolicy: true
      IgnorePublicAcls: true
      RestrictPublicBuckets: true
    CorsConfiguration:
      CorsRules:
        - AllowedHeaders: ["*"]
          AllowedMethods: [GET, PUT, POST, DELETE]
          AllowedOrigins: ["*"]
          MaxAge: 3000
```

The bucket blocks all forms of public access. Images can only be accessed through temporary presigned URLs generated by AWS Lambda (explained in Chapter 5.6).

## Create the AWS Lambda Function and Least-Privilege IAM Role

```yaml
SmartNotesFunction:
  Type: AWS::Serverless::Function
  Properties:
    FunctionName: !Sub "smart-notes-api-${StageName}"
    CodeUri: ./
    Handler: src/lambda.handler
    Runtime: nodejs22.x
    Tracing: Active
    Environment:
      Variables:
        NOTES_TABLE_NAME: !Ref NotesTable
        NOTES_BUCKET_NAME: !Ref NotesBucket
    Policies:
      - DynamoDBCrudPolicy:
          TableName: !Ref NotesTable
      - S3CrudPolicy:
          BucketName: !Ref NotesBucket
    Events:
      RootApi:
        Type: Api
        Properties: { Path: /, Method: ANY }
      ProxyApi:
        Type: Api
        Properties: { Path: /{proxy+}, Method: ANY }
```

`DynamoDBCrudPolicy` and `S3CrudPolicy` are built-in AWS SAM policy templates. They automatically generate IAM policies that grant access only to the referenced DynamoDB table and S3 bucket.

As a result, the Lambda function cannot access any other AWS resources in the account, even if the application contains a security vulnerability.

## Create the Amazon API Gateway

```yaml
SmartNotesApi:
  Type: AWS::Serverless::Api
  Properties:
    StageName: !Ref StageName
    BinaryMediaTypes:
      - "multipart/form-data"
      - "image/*"
    Cors:
      AllowMethods: "'GET,POST,PUT,DELETE,OPTIONS'"
      AllowHeaders: "'Content-Type,Authorization'"
      AllowOrigin: "'*'"
```

## Build and Deploy

```bash
sam build
sam deploy --guided
```

Answer the guided deployment prompts:

```
Stack Name: smart-notes-api
AWS Region: ap-southeast-1
Parameter StageName: dev
Confirm changes before deploy: Y
Allow SAM CLI IAM role creation: Y
Save arguments to configuration file: Y
```

After deployment completes, AWS SAM prints the stack outputs:

```
ApiUrl: https://xxxxxxxxxx.execute-api.ap-southeast-1.amazonaws.com/dev/
NotesTableName: Notes-dev
NotesBucketName: smart-notes-storage-xxxxxxxxxxxx-dev
```

This API endpoint serves all REST routes, including `GET/POST/PUT/DELETE /notes`, image upload and deletion, recycle bin access, and note restoration.

For subsequent deployments after modifying the code, simply run:

```bash
sam build && sam deploy
```
